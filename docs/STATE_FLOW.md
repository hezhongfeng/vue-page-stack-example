# 状态流说明

这个示例项目的重点是演示移动端页面缓存，而不是完整业务系统。

## 核心链路

1. 首页进入 `Home`
2. `Home` 通过底部 tab 切换 `MainList` 和 `About`
3. `MainList` 打开任意 `MainDetail`
4. `MainDetail` 输入内容、继续 push 详情页或跳到 `Login`
5. `Login` 保存用户名并返回
6. 返回后的 `MainDetail` 继续展示缓存输入、激活次数和当前用户

## 状态来源

- 语言状态：`localStorage`
- 登录用户名：`sessionStorage`
- 详情页输入内容：缓存页面实例自身的响应式状态
- 激活次数：缓存页面实例自身的响应式状态
- 页面标题：路由 `meta.titleKey` + `router.afterEach`
- 首页 tab：URL query `tab`

## 推荐验证点

- 切换首页语言后，标题和文案应同步变化
- 进入 `Home?tab=about` 时，标题应显示“关于”
- 详情页输入内容后 push 新页面，再返回，输入仍应存在
- 从详情页跳转到登录页并返回后，当前用户应更新
- 多次重新激活详情页时，激活次数应持续递增

## 测试覆盖

当前单测重点覆盖以下基础能力：

- 语言恢复与持久化
- 路由标题同步
- 登录用户名读写
- 详情页状态组合逻辑

这些测试主要验证状态机制本身，不替代完整的浏览器交互测试。
